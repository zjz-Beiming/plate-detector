# YOLO 车牌检测项目设计文档

## 概述

基于 YOLOv8 的车牌检测桌面应用，支持中国常见车牌（蓝牌、黄牌、绿牌）的实时检测。用户可通过图片、本地视频、摄像头实时流、网络流（RTSP/HTTP）输入进行检测，界面同时显示原始标注图像和裁剪出的车牌区域，并支持保存标注后的结果图片。

技术栈：Python + PyQt6 + Ultralytics YOLOv8 + OpenCV

## 架构设计

3 层分离架构：

- **UI 层（PyQt6）**：左右双栏布局，左侧显示原图+检测框标注，右侧显示裁剪出的车牌缩略图列表，底部为控制栏
- **检测引擎层（YOLOv8）**：加载预训练或自定义模型，执行单帧检测，返回边界框坐标+置信度
- **数据流层**：统合 3 种视频源（本地视频/摄像头/网络流）通过 OpenCV VideoCapture 读取，图片直接加载

核心数据流：输入源 → QThread 异步检测线程 → PyQt6 信号/槽机制推送结果到 UI 主线程 → 左侧显示标注原图，右侧显示裁剪车牌列表。

关键设计决策：
- 检测运行在 QThread 中，避免阻塞 UI 主线程，保证视频流畅
- 通过 PyQt6 的信号/槽机制传递检测结果（边界框坐标、裁剪图像），线程安全
- 视频流使用 OpenCV VideoCapture 统一处理本地视频、摄像头、网络流三种输入
- 视频帧率由检测速度控制：检测完一帧才读取下一帧，避免内存堆积
- 支持暂停/继续：通过线程内的 threading.Event 控制

## UI 组件设计

布局方案：左右双栏

### 4 大组件区域

1. **顶部菜单栏** — 4 种输入源按钮（图片/视频/摄像头/网络流）+ 当前模型名称 + 设置按钮
2. **左侧主画面** — QLabel 显示 OpenCV 图像，叠加检测框和标签（车牌类型+置信度），右上角显示置信度阈值
3. **右侧车牌列表** — 滚动列表，每个车牌项包含：裁剪缩略图 + 颜色分类标签（蓝/黄/绿）+ 置信度百分比 + 检测标记
4. **底部控制栏** — 播放控制 + 进度条 + 保存按钮 + 状态信息（FPS/检测数）

### 按输入模式自适应

| 模式 | 底部控制栏表现 |
|------|----------------|
| 图片 | 隐藏播放控件，只留保存按钮 |
| 视频 | 完整播放控件 + 进度条 + 时间显示 |
| 摄像头 | 播放控件 + 录制时长计时 |
| 网络流 | 播放控件 + 连接状态指示灯 |

## 检测引擎设计

### DetectionEngine 接口

```
DetectionEngine (单例)
├── load_model(model_path)      # 加载 YOLOv8 模型
├── detect(frame) → list[Result] # 单帧检测，返回结果列表
├── set_confidence(threshold)    # 调整置信度阈值
└── classify_plate_color(crop)  # 根据裁剪图判断车牌颜色类别
```

关键设计：
- 使用 ultralytics 库的 YOLOv8 接口，model = YOLO(path) + results = model(frame) 完成检测
- 默认使用预训练车牌检测模型（如基于 CCPD 数据集训练的 YOLOv8 车牌模型），首次运行自动下载；用户也可在设置中指定自定义模型权重文件
- YOLOv8 官方通用模型（coco）不包含中国车牌类别，需使用专门的车牌检测模型
- 车牌颜色分类通过裁剪区域的主色调判定（蓝→蓝牌，黄→黄牌，绿→新能源绿牌）

### 信号定义

```python
detection_finished = pyqtSignal(np.ndarray, list)  # (标注后的帧, 检测结果列表)
plate_cropped = pyqtSignal(list)                    # (裁剪车牌图像列表)
fps_updated = pyqtSignal(float)                     # (当前FPS)
```

## 错误处理策略

| 场景 | 处理方式 |
|------|----------|
| 模型文件不存在 | 启动时自动下载车牌检测模型；下载失败弹窗提示并提供手动选择路径 |
| 视频/图片路径无效 | 弹窗警告 "文件不存在或格式不支持"，恢复到空闲状态 |
| 摄像头打开失败 | 状态栏显示 "摄像头无法打开"，检查设备索引 0→1→2 |
| 网络流连接超时 | 5秒超时，状态栏显示 "连接超时"，提供重试按钮 |
| 网络流中途断开 | 自动重连（最多3次），失败则停止并提示 |
| 检测结果为空 | 左侧正常显示原图（无标注框），右侧显示 "未检测到车牌" 提示文字 |
| 保存路径无权限 | 弹窗提示 "保存失败：权限不足"，引导用户选择其他目录 |

核心原则：任何错误都不能让程序崩溃，所有异常都有对应的用户可见反馈。

## 保存功能

- 点击保存按钮后，将当前左侧显示的标注图（含检测框和标签）保存为图片文件
- 保存格式：JPG（默认）/ PNG（可选）
- 保存路径：用户通过 QFileDialog 选择目录，或使用上次保存的目录
- 文件命名：原始文件名_annotated_时间戳.jpg
